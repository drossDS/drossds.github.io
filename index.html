Boston Airbnb Fair Pricing Tool and Recommender


Developed a pricing model to estimate Boston Airbnb listing prices. Used web scraping, APIs, and natural language processing (NLP) techniques to engineer features. Implemented a recommender system using cosine similarity to suggest listings based on user inputs. The pricing model outperformed the null model by reducing the root mean squared error from $86 to $49 and achieved an R-Squared value of 0.68 on validation data with outliers removed.

Click here to see the code!

...Or click here to expand a brief summary!
Key Accomplishments
  • Data were collected from Inside Airbnb for over 3800 Airbnb listings in the Boston Area in 2023 with 75 features provided for each listing.
  • A feature to simulate the distance of each listing to Boston subway stations was created by web scraping station addresses from the Massachusetts Bay Transit Authority (MBTA) website before using the Google Geocoding API to return latitude and longitude coordinates for each listing.
  • Additional features were engineered which used Natural Language Processing (NLP) Techniques to examine descriptive text columns.
  • The correlation coefficients between all features and listing price were used to select the best features on which to train the model.
  • Seven different regression models were run, and the best models were optimized using grid searching. Principal Component Analysis (PCA) was also attempted to reduce dimensionality and improve price predictions.
  • A simple recommender employing cosine similarity was created to find listings most-similar to simulated user-input listing criteria.
Key Outcomes
  • Models had a large impact on the model performance with the best models initially achieving a root mean squared error (RMSE) of $389 and an R-Squared value of 0.09 on validation data.
  • Performance improved drastically by removing listings with prices below the 5th percentile and above the 95th percentile to an RMSE of $49 and an R-Squared value of 0.68. This is an improvement over the null model’s RMSE of $86 with outliers removed.
  • The best performing model was a random forest regressor.
  • A recommender was capable of correctly recommending listings which met the simulated user-input listing criteria.

Detemining the Severity of Alzheimer's Disease from MRI Images

Built image classification models employing both convolutional and pre-trained neural network models to classify the severity of Alzheimer’s disease in MRI images. The model achieved an accuracy of 71% across four severity classes and improved upon the null model accuracy of 50%. A second binary classification model was developed focusing on detecting very early stages of dementia by training on only two severity classes, achieving an accuracy of 81%, an improvement over the null model accuracy of 59%.

Key Accomplishments
  • Exploratory data analysis (EDA) was performed on images of MRI brain scans obtained from Kaggle. The images were part of four unbalanced classes of dementia severity including “no dementia”, “very mild dementia”, “mild dementia”, and “moderate dementia”.
  • Manually built convolutional neural network (CNN) models were attempted prior to implementing pre-trained image classification models.
  • Binary classification models focusing on improving early detection were created by training some of the previous models exclusively on the no dementia and very mild dementia classes.
Key Outcomes
  • The best model trained on all four dementia classes was the Inception v3 pre-trained achieving an overall accuracy of 71%, outperforming the null model accuracy of 50%.
  • The best early detection binary classification model trained was an optimized CNN model achieving an overall accuracy of 81%, outperforming the binary null model accuracy of 59%.

Identifying Humor with Language Models


Created classification models employing natural language processing to categorize news article titles web scraped from the r/worldnews and r/TheOnion subreddits. Regular expressions were used to clean the data, removing errors and subreddit identifiers in the titles. A variety of classification models were attempted and evaluated. The best performing model was a stacking classifier with an accuracy of 86.0% on unseen data.

Click here to see the code!

...Or click here to expand a brief summary!
Key Accomplishments
  • The Pushshift API was used to collect a total of approximately 6000 news article titles from both the r/worldnews and r/TheOnion subreddits.
  • Natural language processing (NLP) techniques such as tokenization, stop word removal, and sentiment analysis were employed as pre-modeling and exploratory data analysis (EDA) steps.
  • Of seven different classification models initially attempted, the three best models were selected for hyperparameter tuning (optimization) using grid searching.
  • The three best optimized models were then incorporated into a stacking classifier ensemble model - using a logistic regression model as the final estimator - to produce the best results.
Key Outcomes
  • The final model achieved an accuracy of 86% on unseen data, 89% on testing data, and 99.99% on the training data. This is an improvement over the null model accuracy of approximately 51%.
  • It was speculated that the differing subject matter of the two subreddits was likely the reason for such a high accuracy and not due to the model’s ability to correctly identify humor.

Home Price Estimator for Ames, Iowa


Built a linear regression model to predict the price of homes in Ames, Iowa based on home and property characteristics. Engineered interaction and polynomial features to increase model accuracy. The best model achieved an R-Squared value of 0.89 and was able to predict home prices within a mean absolute error of $18,643 and a root mean squared error of $25,887 on testing data.

Key Accomplishments
  • Features were selected based both on their correlations to the home price and common knowledge of important home characteristics that would influence the sale price or value.
  • Different sets of features were run through a linear regression model to find sets of features which would produce the best model performance.
  • Interaction terms and polynomial feature transforms were created and included in the prediction model based on their correlations to price.
  • Regularization techniques were attempted to improve model performance.
Key Outcomes
  • The final model that was selected was trained on a dataset which included polynomial features with a mean absolute error of $18,643, well within the $30,000 goal and an improvement on the null model RMSE of $78,827.
  • Regularization methods produced a slight reduction in model performance with increases in the RMSE below or around 1% of the RMSE from the non-regularized model.
